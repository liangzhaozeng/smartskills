# Publishing Skills

## Overview

Any authenticated user can publish skills to the directory. There are two methods:

## From Git Repo

1. Navigate to `/publish`
2. Select the **From Git Repo** tab
3. Paste your repository URL
4. Fill in the skill name, description, and README
5. Click **Publish Skill**

The skill will be linked to your repo. Users can access the source code directly.

## Direct Upload

1. Navigate to `/publish`
2. Select the **Direct Upload** tab
3. Fill in:
   - **Name**: A descriptive name for your skill
   - **Description**: Brief summary of what the skill does
   - **Category**: e.g., workflow, tools, frontend, backend
   - **Tags**: Comma-separated keywords
   - **Files**: Drop your skill files or click to browse
   - **README**: Markdown documentation
4. Click **Publish Skill**

## Skill Format

A skill typically consists of:

- A markdown file with instructions and prompts
- Optional configuration files
- Optional supporting scripts

## Managing Your Skills

Visit your **Dashboard** to:

- View all your published skills
- See install and click counts
- Edit or delete skills
- Track verification status

## Verification

Admin users can verify skills to indicate they've been reviewed and approved for organization-wide use. Verified skills display a badge on the leaderboard and detail pages.
